# 盈餘修正動能回測 (revision-momentum-backtest)

用台股資料檢驗一個假設:**「分析師共識 EPS / 財測指引被大幅上修的股票,後續 3/6/12 個月相對 0050 有超額報酬。」**

因為 FinMind 免費版**沒有分析師共識/財測資料**,本專案改用兩個「實際數字突然轉強」的**代理訊號**近似「被上修」,並用嚴謹的方法(前視偏誤處理、隨機對照組、樣本內外分段、失敗案例全揭露)去測它到底有沒有 alpha。

> ⚠ 僅供研究,非投資建議。所有數字可由 `data/samples.csv` 與 `cache/` 原始資料重算。

---

## 結論在哪
跑完看 **`backtest_report.md`**。最後一節直接回答:**這個訊號值不值得做成掃描器**(答案是程式依數據算出來的,不是先射箭再畫靶)。

## 假設檢驗方法一覽

| 面向 | 做法 |
| --- | --- |
| 資料 | FinMind 逐檔抓「月營收 + 單季EPS + 日股價」,基準 0050;近 5 年 |
| 代理訊號① REV_ACCEL | 月營收 YoY「加速度」(本月YoY − 上月YoY)≥ `REV_ACCEL_PP`(預設 20pp)且本月YoY>0 |
| 代理訊號② EPS_SURGE | 單季 EPS YoY ≥ `EPS_YOY_PCT`(預設 50%)且去年同季>0 且**超越前四季YoY趨勢** |
| 前視偏誤 | 月營收以「次月10日」、財報以「法定申報期限」為可交易日;進場取**公開日之後**第一個收盤 |
| 輸出 | 勝率、平均超額、最大回撤、樣本數,分 **3/6/12 月** 三個持有期,兩訊號各自 + 合併 |
| 對照組 | 同公開日、同持有期、同股票池的**隨機選股**;edge = 訊號超額 − 對照超額 |
| 誠實報告 | 全部**失敗案例(觸發後照跌)**列表;最慘 40 筆直接秀出來 |
| 過擬合防護 | **只用 2 個可調參數**(上限 3),且採預設值不最佳化;樣本內/外分段;隨機對照 |

## 檔案結構

```
revision-momentum-backtest/
├── params.py            # ★ 所有參數(只有 2 個可調訊號參數)+ 資料範圍設定
├── cache.py             # 檔案快取(讓抓取可中斷可續跑)
├── finmind_client.py    # FinMind 資料層:股票池/月營收/EPS/股價,含限流優雅降級
├── signals.py           # 兩個代理訊號 + 前視偏誤(公開日)處理
├── backtest.py          # 未來報酬/超額/最大回撤 + 指標彙整
├── control.py           # 隨機選股對照組
├── validation.py        # 樣本內/外分段
├── report.py            # 產出 backtest_report.md(含依數據算出的結論)
├── harvest.py           # 背景採集器:跨越限流視窗把資料補齊(無人值守)
├── fetch_data.py        # 一次性抓月營收+EPS(harvest 的簡化版)
├── run.py               # 主流程:訊號→回測→對照→報告
├── data/samples.csv     # 產出:所有樣本(股票×觸發日×持有期×後續報酬)
├── cache/               # API 快取(可刪,刪了會重抓)
└── backtest_report.md   # 產出:最終報告
```

## 怎麼跑

```bash
# 1) 採集資料(逐檔打 FinMind;免費版有每小時上限,故設計成可中斷可續跑)
#    撞到上限會自動睡再續,已抓的都在 cache/。放背景跑、定期看 log:
nohup python3 harvest.py > harvest.log 2>&1 &

# 2) 產報告(可隨時跑;--cached-only 只用已抓到的資料,不打 API、不與採集器搶額度)
python3 run.py --cached-only

# 打開 backtest_report.md 看結論;逐筆樣本在 data/samples.csv
```

- **覆蓋率會隨採集進度提高**:`run.py` 只用「已快取到資料」的股票,`harvest.py` 抓越多、報告樣本越大越穩健。
- **要換設定**:改 `params.py` 的 `REV_ACCEL_PP` / `EPS_YOY_PCT`(訊號),或 `UNIVERSE_SIZE` / `UNIVERSE_SEED`(股票池)。

## 已知限制(務必看報告第 8 節)
免費 API 的三個硬限制,已在報告誠實揭露並說明對結論的影響方向:
1. **無分析師共識** → 用實際數字轉強當代理(最大的代理誤差)。
2. **股價未還原權值**(免費版無 `daily_adj`)→ 有股利拖累,但對照組同口徑部分抵銷。
3. **不能單請求抓全市場** → 用固定種子隨機抽樣代表全市場。
