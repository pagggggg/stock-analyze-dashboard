# 系統總覽 SYSTEM_OVERVIEW

> 這份文件寫給**非工程背景的使用者**。目標是讓你在不看程式的情況下,搞懂這套系統「有什麼、怎麼流動、怎麼用、以及不能拿它做什麼」。
>
> 網站(手機可開):**https://pagggggg.github.io/stock-analyze-dashboard/**
>
> ⚠️ 全系統只用**公開市場數據**做估值與品質研究,**沒有任何持倉或交易紀錄**,**不構成投資建議**。所有數字請回原始財報核實。

## 一、這套系統有哪些模組、各自做什麼

**這在回答什麼問題:「這台機器是由哪些零件組成的?每個零件負責什麼工作?」**

把它想成一條「數字加工廠」的產線。你不用記程式檔名,只要知道有這幾個**功能區塊**:

| 區塊(白話) | 它在做什麼 | 你會怎麼接觸到 |
| --- | --- | --- |
| **資料採購** | 去免費資料源把數字搬回來:公司財報、每天股價、分析師的獲利預估 | 你不用管,背景自動做 |
| **試算引擎** | 用公司法說會的「業績展望」重算台積電的每股盈餘(EPS),分樂觀/中性/悲觀三種劇本 | 主要體現在台積電個股頁 |
| **估值計算** | 把「賺多少」換算成「股價合理值」,並算四個估值指標(前瞻本益比、PEG、自由現金流殖利率、EV/EBITDA) | 個股頁的四張卡片 |
| **現金流體檢** | 檢查「帳上賺的錢有沒有變成真現金」:存貨天數、應收天數、營運現金流趨勢,亮綠/黃/紅燈 | 個股頁的三顆燈 |
| **估值位階圖(河流圖)** | 畫出股價 vs 這檔股票自己過去10年的本益比高低通道,一眼看「現在站在歷史的幾樓」 | 個股頁的河流圖 |
| **訊號偵測器** | 每天比對「今天 vs 昨天」,只挑出**真正的基本面變化**(共識調升/調降、體檢燈變色、估值跨越門檻),**過濾掉每天的股價漲跌雜訊** | 首頁的狀態燈 + 訊號流水 |
| **選股篩選器** | 對一堆股票做兩關卡:第一關排地雷、第二關標優等生,再貼一張估值旗標(貴不貴) | 「選股篩選器」頁 |
| **報告/網頁產生器** | 把上面所有結果,排版成人看得懂的報告和網頁 | 你看到的整個網站 |
| **自動更新與上架** | 每天固定時間自動重跑一次,把最新結果發布到網站 | 你只要開網址就好 |

你實際會「執行」的只有 4 個入口(平常都由自動更新代勞,不必手動):

- `main.py` → 產出**台積電單檔**的詳細估值報告
- `build_site.py` → 產出**整個網站**(首頁 + 各個股頁 + 篩選器頁)
- `fetch_universe.py` → **把全市場資料抓一次存到本地**(篩選器的原料)
- `screen.py` → **跑兩層篩選**,產出選股報告

## 二、完整資料流:從抓資料到產出報告

**這在回答什麼問題:「我按下更新後,數字是從哪裡、經過哪些手,最後變成網頁上那張表的?」**

一句話:**搬數字 → 存起來 → 算一算 → 和昨天比 → 畫成圖表 → 上架**。拆成六步:

```
① 搬數字        免費資料源(FinMind 財報/股價、TWSE 本益比、yfinance 分析師共識/美股)
      │
② 存起來        存進本地檔案 + 快取(cache/),避免重複抓、也能斷網重用
      │
③ 算一算        EPS 三情境、估值四指標、現金流體檢燈、河流圖、篩選器條件、估值旗標
      │
④ 和昨天比      把「今天的快照」對比「上次的快照」→ 只留下訊號級變化(共識升降、燈變色、跨門檻)
      │
⑤ 畫成圖表      排版成 報告(.md) 與 網頁(.html);圖表用 Plotly 內嵌,離線也能看
      │
⑥ 上架          由 GitHub 每天台灣時間 18:00(收盤後)自動重跑,發布到網站
```

**兩個關鍵設計,先講清楚:**

1. **「先手動、再自動」**:每個環節都有「手動備援」。就算某個免費資料源當天壞掉/斷網,系統會自動退回上一版資料或手填檔,**報告照樣產得出來**,不會整個掛掉。
2. **「和昨天比」是重點**:系統每天把每檔股票的狀態存成一張「快照」。**要有昨天的快照,今天才比得出『共識被調升了』這種訊號**。所以自動更新每天都會把最新快照存回專案,長期累積成一條可追蹤的紀錄。

## 三、篩選漏斗全貌:一檔股票怎麼從「全市場」變成「精華清單」

**這在回答什麼問題:「幾千檔股票,系統是用什麼關卡一層層篩,最後留下哪幾檔?每一關到底刷掉了什麼樣的公司?」**

想像一個漏斗,由寬到窄。**核心原則:前兩層只看『公司體質』,最後一層只『貼標籤』,任何一檔都不會因為『貴』而被刪掉。**

```
全市場(台股上市約 2,300 檔)
        │
        ▼  【流動性預篩】先量成交量,太冷門的不深入研究(也省抓取次數)
        │
┌─────────────────────────────────────────────────────────┐
│ 第一關:資格篩選(排地雷)—— 6 條「全部」通過才進池      │
│   ① 上市滿 5 年            (太新、還沒經過景氣考驗的先不看)  │
│   ② 近5年至少4年 EPS 為正   (長期虧損的排除)                │
│   ③ 近3年營運現金流:累積為正 且 至少2年為正 (長期收不到現金的地雷)│
│   ④ 有息負債比 < 門檻(依產業;金融股不看這條) (真槓桿過高的排除)│
│   ⑤ 近60日日均成交金額 > 門檻 (太冷門、進出困難的排除)        │
│   ⑥ 有最新財報(近200天內) (財報難產、資訊不透明的排除)      │
└─────────────────────────────────────────────────────────┘
        │  ← 只要 6 條有任一條沒過(或資料不足),就不進池
        ▼
通過第一層(乾淨、沒有明顯地雷的公司)
        │
┌─────────────────────────────────────────────────────────┐
│ 第二關:品質篩選(標優等生)—— 只標記,不淘汰            │
│   ⑦ 近3年營收 CAGR > 10%      (有沒有在長大)               │
│   ⑧ 近3年毛利率 持平或上升     (賺錢能力有沒有變好)         │
│   ⑨ 近3年 ROE 平均 > 15%       (幫股東賺錢的效率高不高)      │
│   ⑩ 盈餘修正動能(僅標記,不列入判定,等回測驗證後才加權)    │
└─────────────────────────────────────────────────────────┘
        │  ← 第一層通過 + ⑦⑧⑨「三項全達標」= 精華
        ▼
精華清單(體質乾淨 + 品質同時達標)
        │
┌─────────────────────────────────────────────────────────┐
│ 估值旗標層:只貼標籤,不刪任何股 —— 精華再依旗標分三組    │
│   🟢 合理偏低:PEG<1 且 前瞻本益比 < 該股自己近5年本益比中位  │
│   🟡 一般                                                  │
│   🔴 高估值警戒:前瞻本益比 > 該股自己近5年的90百分位,       │
│                 或 PEG>2,或 前瞻本益比>60倍                 │
└─────────────────────────────────────────────────────────┘
        │
        ▼
   🟢 精華+綠旗(優先研究)   🟡 精華+黃旗   🔴 精華+紅旗(好公司但貴,附警語)
```

**每一關實際會刷掉什麼(用目前 8 檔測試的真實結果舉例):**

- **③ 現金流**:刷掉了**廣達**——它近3年營運現金流有兩年是負的(狂擴 AI 伺服器、現金被存貨與應收吃掉)。這正是要抓的「長期收不到現金」地雷,而不是單季波動。
- **④ 有息負債比**:改用「真槓桿」口徑後,**鴻海、瑞昱、廣達、日月光全部平反**(它們的「負債」大多是應付帳款,不是真借款)。舊的「總負債比」會冤殺代工與 IC 設計股。
- **第二層品質**:台積電、台達電三項全達標 → 進**精華**;聯發科(景氣淡年成長掉)、中華電(低成長、ROE 不足)雖然體質乾淨,但品質沒全達標,不進精華。
- **🔴 紅旗**:聯發科(前瞻本益比 58 倍,遠高於自己歷史)、中華電(PEG 6.8)、日月光、TSLA(本益比 150 倍)都被貼紅旗——**但沒有被刪掉**,只是附上警語提醒。

> ⚠️ 現況提醒:目前篩選器跑的是 **8 檔台股 + TSLA(測試)**,**全市場尚未實跑**(見第六節)。上面的 2,300 檔是全市場放大後的規模,漏斗邏輯一樣。

## 四、我該怎麼使用:日常 / 每週 / 每季

**這在回答什麼問題:「我不是每天都有時間,到底該在什麼時候、看哪一頁、依什麼順序看,才不會被雜訊淹沒?」**

原則:**看得越勤,越要只看『訊號』;真正花時間研究,放在每週與每季。**

### 🟢 每天(30 秒,只看有沒有事)
1. 打開網站首頁,只看最上面那顆**狀態燈**:
   - 🟢 綠 = 無訊號級變化 → **關掉,今天沒事**。
   - 🟡 黃 = 有共識調升/調降,或某檔現金流體檢燈變色 → 往下看「訊號流水」那幾行。
   - 🔴 紅 = 有股票跨越估值門檻 → 看是哪一檔、怎麼跨的。
2. **不要**因為「今天某檔漲/跌了 3%」就行動——股價漲跌**刻意不列入訊號流水**,那是雜訊。

### 🟡 每週(10 分鐘,縮小研究範圍)
1. 看**掃描總表**:所有觀察股的四個估值指標一覽,點欄位可排序。這是「縮小範圍」用,不是買進清單。
2. 看**選股篩選器頁**:
   - 先看**精華清單的三組**:🟢 綠旗組是「品質好又相對不貴」→ **優先研究**;🔴 紅旗組是「好公司但貴」→ 讀那段紅旗警語,別急著追。
   - 對有興趣的個股,點進**個股頁**看:河流圖(現在站在歷史估值幾樓)、現金流三顆燈(賺的是不是真現金)。

### 🟣 每季(財報/法說後,30–60 分鐘,更新假設)
1. 台積電法說會後,打開 `config/assumptions.yaml`,把新的「業績展望」數字(季營收、毛利率、匯率)換上去,每個數字都寫上來源。
2. 重跑報告,看第六節的**模型回測**:同一套公式套回過去幾季,和財報實際 EPS 差多少——差越小,代表這季的試算越可信。
3. 看**共識監控**:分析師對今年/明年的獲利預估,這季被**調升還是調降**——這比股價波動更值得花時間研究背後原因。
4. (可選)想擴大選股範圍時,跑一次 `fetch_universe.py` 更新全市場資料,再 `screen.py`。

**一句話順序**:每天看**燈** → 每週看**篩選器精華組 + 個股河流圖** → 每季看**回測 + 共識升降**。

## 五、這套系統「不能做什麼」——邊界與已知限制

**這在回答什麼問題:「哪些事我以為它會做、但其實它做不到?我不該拿它當什麼用?」**

**請把這一節當成使用前的警語,逐條讀完。**

1. **它不會告訴你「買/賣/持有」,也不預測股價。** 它只重建數字、貼標籤、標訊號。所有「要不要進場、進多少」都得你自己決定。
2. **它不含任何持倉、成本、損益。** 這是一套**純研究工具**,不知道你買了什麼、賺賠多少,也不該拿來記帳。
3. **估值旗標不是買賣訊號。** 🟢 綠旗只代表「相對它自己歷史不算貴」,不代表「該買」;🔴 紅旗不代表「該賣」,只代表「現價已隱含很高的成長預期,買前要想清楚」。
4. **通過篩選 ≠ 好投資。** 篩選器只排「明顯地雷」+ 標「品質指標」,它**看不懂**:經營團隊好壞、護城河、產業結構轉變、地緣政治、一次性事件、財報造假。這些都要你自己補。
5. **分析師共識可能是錯的。** PEG、預期差、共識監控都建立在「分析師預估」上;預估會失準,尤其在轉折點。系統只如實呈現,不保證它對。
6. **模型回測是「自我檢查」,不是「預測未來」。** 回測只證明「這套公式套回過去算得準」,不保證未來的試算會準。
7. **它不做投資組合、風險、相關性、部位配置。** 沒有「該買幾%、如何分散、停損設哪」這類功能。
8. **歷史本益比只跟『自己』比,不跨股比貴賤。** 河流圖與估值旗標的百分位,用的是**個股自己的歷史**——不同產業的合理本益比天生不同,不能拿台積電的倍數去套中華電。
9. **不是即時報價系統。** 資料每天收盤後更新一次;盤中不會即時變動。
10. **僅涵蓋台股(上市)與少數美股測試標的。** 上櫃、興櫃、其他市場、ETF、權證都不在範圍內。

## 六、目前已知的資料缺口與待修項目

**這在回答什麼問題:「這套系統現在有哪些地方還沒做完、或資料本身就有洞?我看報告時要對哪些數字打個問號?」**

**誠實揭露,依重要性排序:**

| 項目 | 現況 / 缺口 | 影響 | 打算怎麼處理 |
| --- | --- | --- | --- |
| **全市場尚未實跑** | 篩選器目前只跑 **8 檔台股 + TSLA(測試)** | 精華清單還不是「全市場」的結論 | 確認門檻合理後,跑一次 `fetch_universe.py`(免費額度下約數小時,建議先設 FinMind token) |
| **美股財報年數不足** | yfinance 免費只給約 4–5 個完整年度;TSLA 最舊一年是空的 | 美股的「近5年 EPS」這條常會標「⚠️資料不足」→ 不當通過(誠實,但也讓部分美股卡關) | 之後接更完整的美股財報源,或對美股改用 4 年門檻(需你確認) |
| **台股分析師共識覆蓋率低** | yfinance 對部分台股(尤其金融、消費股)只回 1 家分析師,實際覆蓋更多 | 覆蓋家數不可靠 → 硬卡「≥3家」會誤刪好股 | **已處理(不刪只標記)**:母體門檻降到 ≥1、守門靠法說會+市值+流動性;覆蓋<3 者在清單與下游 PEG/修正動能標「⚠低覆蓋·僅供參考」。**待辦**:未來接更可靠的台股覆蓋源(現在先以跑全市場為優先) |
| **修正動能尚未納入評分** | 第⑩條「共識調升」目前**只標記、不加權** | 它不影響「兩層全過」判定 | **刻意保留**:等 `revision-momentum-backtest/` 子專案回測驗證「調升是否真的有預測力」後,才決定加權重 |
| **本益比百分位是「前瞻 vs 歷史」混用口徑** | 旗標拿「前瞻本益比」去比「歷史的過去式本益比」分布 | 成長股的前瞻本益比通常較低,可能讓它看起來偏便宜一點 | 已在報告註明;看**相對高低與趨勢**即可,別當精準門檻 |
| **美股歷史本益比是粗估** | 美股用「年度 EPS」近似每日本益比(免費資料只有年度) | 美股的近5年中位/P90 較粗糙 | 屬測試階段近似;台股用每日 EPS 較精細 |
| **「上市滿5年」用財報年數代理** | 用「最早一筆財報」推算,而非真正上市掛牌日 | 極少數情況可能誤判一兩年 | 影響小;必要時改接掛牌日資料 |
| **雲端自動更新偶爾受第三方限流** | GitHub 雲端跑 yfinance 偶爾被 Yahoo 擋 | 該次某些估值/共識欄會顯示 N/A(財報/股價的 FinMind 較穩,是主幹) | 有重試與退回機制;通常隔天恢復 |
| **綠旗精華目前是空的** | 8 檔測試中,品質達標者剛好都不便宜 | 「優先研究」組暫時沒東西 | 放大到全市場、或加入品質好又便宜的標的後自然會有 |
